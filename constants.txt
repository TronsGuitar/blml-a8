# VB6 Built-in Constants Reference

Below is a comprehensive list of the built-in constants in Visual Basic 6.0, organized by category. Each constant’s name is given along with its value (in decimal or hex) and a brief description.

## File I/O Constants  
These constants are used for file handling operations such as `Open`, `GetAttr`, `Dir`, etc.:

- **File Open Mode constants** – Used in the `Open ... For ...` statement to specify mode of file access:  
  - `vbInput` (1) – Open file for **Input** (read) ([ [VB6]Open files using the Script Control's ExecuteStatement method-VBForums](https://www.vbforums.com/showthread.php?691019-VB6-Open-files-using-the-Script-Control-s-ExecuteStatement-method&p=4235163#:~:text=,vbBinary%20%3D%2032))  
  - `vbOutput` (2) – Open file for **Output** (write, truncating existing data) ([ [VB6]Open files using the Script Control's ExecuteStatement method-VBForums](https://www.vbforums.com/showthread.php?691019-VB6-Open-files-using-the-Script-Control-s-ExecuteStatement-method&p=4235163#:~:text=,vbBinary%20%3D%2032))  
  - `vbRandom` (4) – Open file for **Random access** (read/write fixed-length records) ([ [VB6]Open files using the Script Control's ExecuteStatement method-VBForums](https://www.vbforums.com/showthread.php?691019-VB6-Open-files-using-the-Script-Control-s-ExecuteStatement-method&p=4235163#:~:text=,vbBinary%20%3D%2032))  
  - `vbAppend` (8) – Open file for **Append** (write at end of file) ([ [VB6]Open files using the Script Control's ExecuteStatement method-VBForums](https://www.vbforums.com/showthread.php?691019-VB6-Open-files-using-the-Script-Control-s-ExecuteStatement-method&p=4235163#:~:text=,vbBinary%20%3D%2032))  
  - `vbBinary` (32) – Open file in **Binary** mode (read/write binary data) ([ [VB6]Open files using the Script Control's ExecuteStatement method-VBForums](https://www.vbforums.com/showthread.php?691019-VB6-Open-files-using-the-Script-Control-s-ExecuteStatement-method&p=4235163#:~:text=,vbBinary%20%3D%2032))

- **File Attribute constants** – Used with functions like `GetAttr`, `SetAttr`, and `Dir` to work with file attributes ([GetAttr function (Visual Basic for Applications) | Microsoft Learn](https://learn.microsoft.com/en-us/office/vba/language/reference/user-interface-help/getattr-function#:~:text=Constant%20Value%20Description%20vbNormal%200,Not%20available%20on%20the%20Macintosh)) ([GetAttr function (Visual Basic for Applications) | Microsoft Learn](https://learn.microsoft.com/en-us/office/vba/language/reference/user-interface-help/getattr-function#:~:text=vbNormal%200%20Normal,Not%20available%20on%20the%20Macintosh)):
  - `vbNormal` (0) – Normal file (no special attributes) ([GetAttr function (Visual Basic for Applications) | Microsoft Learn](https://learn.microsoft.com/en-us/office/vba/language/reference/user-interface-help/getattr-function#:~:text=Constant%20Value%20Description%20vbNormal%200,Not%20available%20on%20the%20Macintosh))  
  - `vbReadOnly` (1) – Read-only file ([GetAttr function (Visual Basic for Applications) | Microsoft Learn](https://learn.microsoft.com/en-us/office/vba/language/reference/user-interface-help/getattr-function#:~:text=Constant%20Value%20Description%20vbNormal%200,Not%20available%20on%20the%20Macintosh))  
  - `vbHidden` (2) – Hidden file ([GetAttr function (Visual Basic for Applications) | Microsoft Learn](https://learn.microsoft.com/en-us/office/vba/language/reference/user-interface-help/getattr-function#:~:text=Constant%20Value%20Description%20vbNormal%200,Not%20available%20on%20the%20Macintosh))  
  - `vbSystem` (4) – System file ([GetAttr function (Visual Basic for Applications) | Microsoft Learn](https://learn.microsoft.com/en-us/office/vba/language/reference/user-interface-help/getattr-function#:~:text=vbNormal%200%20Normal,Not%20available%20on%20the%20Macintosh))  
  - `vbDirectory` (16) – Directory/folder ([GetAttr function (Visual Basic for Applications) | Microsoft Learn](https://learn.microsoft.com/en-us/office/vba/language/reference/user-interface-help/getattr-function#:~:text=vbReadOnly%201%20Read,Not%20available%20on%20the%20Macintosh))  
  - `vbArchive` (32) – Archive file (file has changed since last backup) ([GetAttr function (Visual Basic for Applications) | Microsoft Learn](https://learn.microsoft.com/en-us/office/vba/language/reference/user-interface-help/getattr-function#:~:text=vbSystem%204%20System%20file,Not%20available%20on%20the%20Macintosh))  
  - `vbAlias` (64) – Alias (Macintosh only) ([Dir function (Visual Basic for Applications) | Microsoft Learn](https://learn.microsoft.com/en-us/office/vba/language/reference/user-interface-help/dir-function#:~:text=vbSystem%204%20Specifies%20system%20files,Available%20only%20on%20the%20Macintosh))  

*(Use bitwise operations to test or combine file attributes. For example, `If GetAttr(fname) And vbHidden Then ...` checks if a file is hidden.)*

## Message Box Constants  
When calling the `MsgBox` function, you can use these constants to specify the buttons, icons, default button, modality, and return values ([ MsgBox and flags - please about explanation-VBForums](https://www.vbforums.com/showthread.php?447289-MsgBox-and-flags-please-about-explanation)) ([ MsgBox and flags - please about explanation-VBForums](https://www.vbforums.com/showthread.php?447289-MsgBox-and-flags-please-about-explanation#:~:text=,button%20to%20the%20message%20box)):

- **Button combination constants** (for the first argument of `MsgBox`, values 0–5):  
  - `vbOKOnly` (0) – OK button only ([ MsgBox and flags - please about explanation-VBForums](https://www.vbforums.com/showthread.php?447289-MsgBox-and-flags-please-about-explanation))  
  - `vbOKCancel` (1) – OK and Cancel buttons ([ MsgBox and flags - please about explanation-VBForums](https://www.vbforums.com/showthread.php?447289-MsgBox-and-flags-please-about-explanation))  
  - `vbAbortRetryIgnore` (2) – Abort, Retry, and Ignore buttons ([ MsgBox and flags - please about explanation-VBForums](https://www.vbforums.com/showthread.php?447289-MsgBox-and-flags-please-about-explanation))  
  - `vbYesNoCancel` (3) – Yes, No, and Cancel buttons ([ MsgBox and flags - please about explanation-VBForums](https://www.vbforums.com/showthread.php?447289-MsgBox-and-flags-please-about-explanation))  
  - `vbYesNo` (4) – Yes and No buttons ([ MsgBox and flags - please about explanation-VBForums](https://www.vbforums.com/showthread.php?447289-MsgBox-and-flags-please-about-explanation#:~:text=,48%20Display%20Warning%20Message%20icon))  
  - `vbRetryCancel` (5) – Retry and Cancel buttons ([ MsgBox and flags - please about explanation-VBForums](https://www.vbforums.com/showthread.php?447289-MsgBox-and-flags-please-about-explanation#:~:text=,48%20Display%20Warning%20Message%20icon))

- **Icon constants** (adds an icon to the message box, values 16, 32, 48, 64):  
  - `vbCritical` (16) – Critical Error icon (stop sign) ([ MsgBox and flags - please about explanation-VBForums](https://www.vbforums.com/showthread.php?447289-MsgBox-and-flags-please-about-explanation#:~:text=,64%20Display%20Information%20Message%20icon))  
  - `vbQuestion` (32) – Question Mark icon (for prompts) ([ MsgBox and flags - please about explanation-VBForums](https://www.vbforums.com/showthread.php?447289-MsgBox-and-flags-please-about-explanation#:~:text=,64%20Display%20Information%20Message%20icon))  
  - `vbExclamation` (48) – Exclamation Point icon (warning) ([ MsgBox and flags - please about explanation-VBForums](https://www.vbforums.com/showthread.php?447289-MsgBox-and-flags-please-about-explanation#:~:text=,64%20Display%20Information%20Message%20icon))  
  - `vbInformation` (64) – Information icon (lower-case “i”) ([ MsgBox and flags - please about explanation-VBForums](https://www.vbforums.com/showthread.php?447289-MsgBox-and-flags-please-about-explanation#:~:text=,First%20button%20is%20the%20default))

- **Default button constants** (choose which button is focused by default, add 256 * (n-1)):  
  - `vbDefaultButton1` (0) – First button is the default ([ MsgBox and flags - please about explanation-VBForums](https://www.vbforums.com/showthread.php?447289-MsgBox-and-flags-please-about-explanation#:~:text=,Fourth%20button%20is%20the%20default))  
  - `vbDefaultButton2` (256) – Second button default ([ MsgBox and flags - please about explanation-VBForums](https://www.vbforums.com/showthread.php?447289-MsgBox-and-flags-please-about-explanation#:~:text=,Fourth%20button%20is%20the%20default))  
  - `vbDefaultButton3` (512) – Third button default ([ MsgBox and flags - please about explanation-VBForums](https://www.vbforums.com/showthread.php?447289-MsgBox-and-flags-please-about-explanation#:~:text=,Fourth%20button%20is%20the%20default))  
  - `vbDefaultButton4` (768) – Fourth button default ([ MsgBox and flags - please about explanation-VBForums](https://www.vbforums.com/showthread.php?447289-MsgBox-and-flags-please-about-explanation#:~:text=,work%20in%20the%20current%20application))

- **Modality constants** (application vs system modal):  
  - `vbApplicationModal` (0) – Application-modal (user must dismiss message box before continuing in *current* app) ([ MsgBox and flags - please about explanation-VBForums](https://www.vbforums.com/showthread.php?447289-MsgBox-and-flags-please-about-explanation#:~:text=,button%20to%20the%20message%20box))

- **Additional MsgBox style flags** (can be combined as needed):  
  - `vbMsgBoxHelpButton` (16384) – Adds a Help button to the message box ([ MsgBox and flags - please about explanation-VBForums](https://www.vbforums.com/showthread.php?447289-MsgBox-and-flags-please-about-explanation#:~:text=,message%20box%20window%20as%20the))

- **Return value constants** (for the function result of `MsgBox`, indicating which button was clicked):  
  - `vbOK` (1) – OK button clicked ([List of VBScript constants - StealthBot Wiki](https://sbwiki.davnit.net/index.php/List_of_VBScript_constants#:~:text=These%20specify%20the%20result%20of,which%20button%20the%20user%20clicked))

  - `vbAbort` (3) – Abort clicked ([List of VBScript constants - StealthBot Wiki](https://sbwiki.davnit.net/index.php/List_of_VBScript_constants#:~:text=%2A%20%60vbOK%60%20%281%29%20,No%20was%20clicked))  
  - `vbRetry` (4) – Retry clicked ([List of VBScript constants - StealthBot Wiki](https://sbwiki.davnit.net/index.php/List_of_VBScript_constants#:~:text=%2A%20%60vbCancel%60%20%282%29%20,No%20was%20clicked))  
  - `vbIgnore` (5) – Ignore clicked ([List of VBScript constants - StealthBot Wiki](https://sbwiki.davnit.net/index.php/List_of_VBScript_constants#:~:text=%2A%20%60vbCancel%60%20%282%29%20,No%20was%20clicked))  
  - `vbYes` (6) – Yes clicked ([List of VBScript constants - StealthBot Wiki](https://sbwiki.davnit.net/index.php/List_of_VBScript_constants#:~:text=%2A%20%60vbAbort%60%20%283%29%20,No%20was%20clicked))

*(*Usage:* These constants can be **OR**’ed together to form the `Buttons` argument in MsgBox. For example: `MsgBox "Text", vbYesNo + vbQuestion + vbDefaultButton2`).*

## Error Constants  
Visual Basic 6 defines a constant to help create user-defined errors:

- `vbObjectError` – Value **-2147221504** (hex 0x80040000) ([List of VBScript constants - StealthBot Wiki](https://sbwiki.davnit.net/index.php/List_of_VBScript_constants#:~:text=And%20other%20miscellaneous%20constants)). This is used as a base value when defining custom error numbers in VB. For example, to raise a custom error, one might do:  
  `Err.Raise vbObjectError + 1000, , "My custom error message"` – which produces an error in the user-defined error range ([Q189134: HOWTO: Raise an Error in Visual Basic From Your C DLL](https://www.infania.net/misc/kbarchive/kb/189/Q189134/index.html#:~:text=DLL%20www,So%20when%20you%20raise)).

*(VB does not provide named constants for every error code, but the `Err` object’s properties (Number, Description, etc.) are used to handle runtime errors. The constant above helps in avoiding conflicts with VB’s internal error numbers by shifting custom errors into a reserved range.)*

## Registry Constants  
Visual Basic itself doesn’t have native constants for registry access (since VB6 uses the `GetSetting` and `SaveSetting` functions for VB-specific registry paths and typically relies on Windows API or WMI for general registry tasks). However, when working with the Windows registry via APIs or scripting, it is common to use predefined constant values for the registry root keys (HKEYs). These are often defined in code as: 

- `HKEY_CLASSES_ROOT` = &H80000000 ([Managing Windows Registry with Scripting (Part 1) | ServerWatch](https://www.serverwatch.com/guides/managing-windows-registry-with-scripting-part-1/#:~:text=Const%20HKEY_CLASSES_ROOT%20%20%20,HKEY_CURRENT_CONFIG%20%20%20%3D%20%26H80000005))  
- `HKEY_CURRENT_USER` = &H80000001 ([Managing Windows Registry with Scripting (Part 1) | ServerWatch](https://www.serverwatch.com/guides/managing-windows-registry-with-scripting-part-1/#:~:text=Const%20HKEY_CLASSES_ROOT%20%20%20,HKEY_CURRENT_CONFIG%20%20%20%3D%20%26H80000005))  
- `HKEY_LOCAL_MACHINE` = &H80000002 ([Managing Windows Registry with Scripting (Part 1) | ServerWatch](https://www.serverwatch.com/guides/managing-windows-registry-with-scripting-part-1/#:~:text=Const%20HKEY_CLASSES_ROOT%20%20%20,HKEY_CURRENT_CONFIG%20%20%20%3D%20%26H80000005))  
- `HKEY_USERS` = &H80000003 ([Managing Windows Registry with Scripting (Part 1) | ServerWatch](https://www.serverwatch.com/guides/managing-windows-registry-with-scripting-part-1/#:~:text=Const%20HKEY_LOCAL_MACHINE%20%20%20,HKEY_CURRENT_CONFIG%20%20%20%3D%20%26H80000005))  
- `HKEY_CURRENT_CONFIG` = &H80000005 ([Managing Windows Registry with Scripting (Part 1) | ServerWatch](https://www.serverwatch.com/guides/managing-windows-registry-with-scripting-part-1/#:~:text=Const%20HKEY_LOCAL_MACHINE%20%20%20,HKEY_CURRENT_CONFIG%20%20%20%3D%20%26H80000005))

Additionally, if working with the registry, you might encounter constants for data types (from the Windows API), for example: `REG_SZ` = 1 (string value), `REG_DWORD` = 4 (32-bit number), etc., and others like `KEY_READ`, `KEY_WRITE` access flags. These are not built into VB6, but can be declared as needed when calling registry APIs ([How to read a registry key and its values (VBScript) - Sysadmins LV](https://www.sysadmins.lv/retired-msft-blogs/alejacma/how-to-read-a-registry-key-and-its-values-vbscript.aspx#:~:text=LV%20www,Const)).
